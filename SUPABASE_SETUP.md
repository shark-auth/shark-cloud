# Supabase Configuration Guide for SharkAuth

This guide outlines the steps to configure your Supabase backend to support the SharkAuth landing page features: Waitlist, Demo Scheduling, and Telemetry.

## 1. Create Tables

Run the following SQL in the Supabase SQL Editor:

```sql
-- 1. Waitlist Table
CREATE TABLE public.waitlist (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
  email TEXT NOT NULL UNIQUE
);

-- 2. Schedule Demo Table
CREATE TABLE public.schedule_demo (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
  first_name TEXT,
  last_name TEXT,
  email TEXT NOT NULL,
  use_case TEXT
);

-- 3. Telemetry Table
CREATE TABLE public.telemetry (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
  installation_id TEXT NOT NULL
);

-- Enable RLS (Row Level Security) but allow Service Role or Anon access if needed
-- Note: For simple lead capture, you might want to allow PUBLIC insert but no read.
ALTER TABLE public.waitlist ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.schedule_demo ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.telemetry ENABLE ROW LEVEL SECURITY;

-- Create policies for public insertion
CREATE POLICY "Enable insert for everyone" ON public.waitlist FOR INSERT WITH CHECK (true);
CREATE POLICY "Enable insert for everyone" ON public.schedule_demo FOR INSERT WITH CHECK (true);
CREATE POLICY "Enable insert for everyone" ON public.telemetry FOR INSERT WITH CHECK (true);
```

## 2. Environment Variables

Add the following to your `.env.local` file (and your production environment):

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

## 3. Verify Integration

You can test the endpoints using `curl`:

### Waitlist
```bash
curl -X POST http://localhost:3000/api/waitlist \
  -H "Content-Type: application/json" \
  -d '{"email": "test@example.com"}'
```

### Schedule Demo
```bash
curl -X POST http://localhost:3000/api/schedule \
  -H "Content-Type: application/json" \
  -d '{"firstName": "John", "lastName": "Doe", "email": "john@example.com", "useCase": "Agent Authentication"}'
```

### Telemetry
```bash
curl -X POST http://localhost:3000/api/telemetry \
  -H "Content-Type: application/json" \
  -d '{"installation_id": "abc-123-xyz"}'
```
